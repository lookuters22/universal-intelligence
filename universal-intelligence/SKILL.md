# Universal Intelligence (UI) Ruleset



In this repo **UI = Universal Intelligence** (this documented method). Third-party trademarks or unrelated product lines live outside these files.



## Purpose



### What AI reliably is (creative / problem-solving work)



**Higher bar of ordinary.** Completion models can polish output so it reads *fine* yet stays **median**—better than empty page, worse than memorable. Mechanism: probability mass concentrates on consensus patterns; if the same shortcut runs **for you and for everyone**, advantage diffuses—you only lift the **baseline of interchangeable adequacy**. Net: mediocre = not evil, merely **average, ordinary, undistinguished**.



**New status quo = still status quo.** The model may nudge envelopes, yet when it executes the drift **for everyone the same way**, that drift becomes tomorrow’s cliché—you never escape ordinariness by outsourcing the exploratory path wholesale.



**It will not sit in wrong.** By default assistants optimize coherence with dominant priors—they do not inhabit sustained, load-bearing exploratory **wrong-road** deliberately the way humans (or disciplined UI procedure) must.



**Under the hood = no apprenticeship.** Outputs land **without procedural witness** comparable to mixing behind a guarded console—you cannot watch decisions land, replay fork points, rehearse rationale. Observation collapses ⇒ skill plateau unless you forcibly dissect.



Treat that bundle as gravity toward **7/10 normal**.



### What UI operationalizes here



**Exploratory wrongness with human reins.** Creativity worth the name disproportionately nests in **ocean-of-wrong** passes: jump off correctness cliff intentionally, generate reviewable exploratory mass, scout friction, quarries, and hinges—not random sabotage, not spectacle entropy.



Pipeline (domain-agnostic): **garbage exploratory volume → meticulous review noting fascinating wrong fragments → pivot/quarry assembling an end artifact whose journey would have been vetoed if you marched the orthodox roadmap from hour zero.** A competent craftsperson faithfully following “correct-first” choreography often **never generates that road**.



**Velocity without surrendering cockpit.** Aspire to brisk iteration like automation grants—**yet** expose mechanism, checkpoints, rationales alternatives killed so the runner **learns** each cycle (contrast unattended black-box polishing).



**Teaching mandate attaches to final emits:** deliverables must improve the user’s model of the craft—not solely deliver camouflaged answers.



You still honor the strict four-phase workflow; exploratory entropy stays **replayable audition material**, not hollow shock.



Abort when output relapses undistinguished cliché lacking earned pivot lineage.



### Portable analogy (no fixed domain)



Rookie wildly composes contradictory frameworks into a clumsy spike; senior reviews, isolates asymmetric shards, rewires routing or policy architecture around salvaged quirks. Finished design **signals** ancestry from nonsense path—you could not have shipped identical personality by insisting correct-first scaffolding day one.



Carry analogue across software, org design, rhetoric, bargaining, ethics sketches—any brainstorming or diagnostic prompt where rules allow speculative structure.



## Precedence



When rules conflict:



1. Truth, safety, accessibility, laws, abuse prevention, credential handling, cryptographic correctness, medically sensitive claims.

2. Explicit user constraints.

3. The 4-step UI loop executed **in sequence** with **no lookahead** across deliverables belonging to future steps.

4. The mandate to leave the trough when the user's task welcomes ideation, **problem-solving**, architecture forks, naming, narratives, or speculative product logic (never as a loophole against item **1**).

5. Core rules and heuristics below.



## Scope



Universal application: brainstorming, diagnostics, dispute resolution sketches, UX/product/process design, naming, narratives, organizational moves, reversible experiments—and **architecture / codebase forks** once domain rules allow speculation. Never a waiver for mandated correctness (payments, HIPAA, aerospace, cryptography as specified).



### Product / market gap mode



When the user asks for product ideas, startup ideas, business opportunities, market gaps, unsaturated markets, monetizable wedges, competitor avoidance, or "ideas that can win," run `references/market-gap-research.md` and `references/refusal-layer.md` before the four-phase UI loop.



This is a **pre-flight evidence loop**, not a shortcut to a final answer. Research current alternatives, map saturation, form gap hypotheses, then actively try to kill those hypotheses with counter-search. Route nothing forward until the refusal layer classifies the candidate as `ROUTE`. Repeat until one candidate survives as high-conviction and testable, or until evidence proves the market/path is too saturated or weak and the loop must restart around a sharper segment.



Do **not** claim literal 100% certainty. Interpret requests for a "100% good idea" as a demand for a genuinely useful, evidence-backed, low-delusion idea with a named riskiest assumption, kill criteria, and first validation probe.



## Core workflow — strict sequence



Finish each phase before borrowing later-phase language or structure.



| Step | Name | Forbidden while in this step |

|------|------|-------------------------------|

| 1 | The Forest of Wrong | Naming polished final SKU, phased rollout KPIs, "MVP roadmap," prematurely sanding wrong passes toward bland respectability |

| 2 | Friction Analysis | Selecting a winner, applying inversion equations / harvest moves; merging exploratories into one safe stew |

| 3 | The Pivot | Full production story that erases exploratory DNA; rewriting history as if Step 1 always obeyed textbooks |

| 4 | Refinement | Retro-fitting tame answers then inventing exploratories to fake compliance |



Restart from Step 1 if **The Relapse Test** fails.



### Step summary



0. **Market Gap Loop** *(only for product / market / business ideation)*: current research, saturation map, adversarial doubt pass, and survivor ledger (`references/market-gap-research.md`).

0.5. **Evidence Refusal Layer** *(only for product / market / business ideation)*: refuse, narrow, or rerun weak findings before they can feed Step 1 (`references/refusal-layer.md`).

1. **The Forest of Wrong**: Dogma mapping then 4–5 **Garbage Exploratories** / load-bearing passes (`references/the-forest-of-wrong.md`).

2. **Friction Analysis**: **Engineer-review stance** — each exploratory → somatic + systemic friction + Pain Coordinate (+ optional Grain note); locate fascinating wrong fragments (`references/friction-analysis.md`). No fixes.

3. **The Pivot**: One cluster elevated into mechanism / quarry canon **the correct road seldom reaches** (`references/the-pivot.md`).

4. **Refinement**: Actionable artifact with **transparent control surfaces + teaching payload** (`references/refinement-heuristics.md`).



## The Relapse Test



Internally:



> Stripped labels, does this resemble generic SaaS / CRUD / roadmap lint, tepid checklist wisdom, textbook best-practice obtainable **without** wrong-road exploration + disciplined pivot/quarry?

>

> Could a careful operator **replay why** this beats ordinary—witness forks, rationales—or is it guarded-under-hood seasoning?



Affirm hiding black box ⇒ fail. Restart Step 1 with deeper exploratory families—not tonal edits.



Deliverable retains **ancestry of misassignment / quarried shard** visibly operative—not wallpaper.



## Core rules



### 1. No premature respectable answers



Erase first mainstream instinct before Step 4.



### 2. Serious exploratory entropy



Hermetic hostility default off. Garbage passes must resemble **honest exploratory bulk** salvageable corners unless domain screams adversarial guard rails.



### 3. Honor engineer-review stance



Friction analysis hunts **interesting wrong fragments**, not etiquette fixes prematurely.



### 4. Pivot / quarry truth



Maintain lineage misassignment → friction → powering hinge / quarry canon. Quiet averaging taboo.



### 5. Legible imperative delivery



Structured blunt prose—strip hollow assistant ceremony.



### 6. Pseudoconformity / spectacle bans



Banned cartoon chaos masking tidy core; banned pure-noise blobs untethered Grain (`references/anti-patterns.md`).



### 7. Human cockpit + pedagogical payoff



Prefer inspectable rationales, reproducible checkpoints, surfaced trade-offs—explicit **why this beats ordinary**. Default forbid finish that merely **does the thinking under the hood** without upgrading user mental model.



### 8. Non-relapse originality claim



Articulate why orthodox roadmap probably never reaches same signature—earn boldness.
